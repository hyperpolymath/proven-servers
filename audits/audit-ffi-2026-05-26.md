# Audit: FFI unsafe blocks (proven-servers)

**Auditor**: Jonathan D.A. Jewell  
**Date**: 2026-05-26  
**Scope**: panic-attack assail Critical/High UnsafeCode/UnsafeFFI findings under `bindings/rust/src/`.  
**Registry**: `audits/assail-classifications.a2ml`.

## Rationale

Rust FFI bindings (`ffi_smtp.rs`, `ffi_ftp.rs`, `ffi_grpc.rs`, etc.) to libproven-servers' C ABI. Underlying logic is formally-verified Idris2. Every unsafe block sits at the C ABI boundary and is required by Rust to call across.

Refs hyperpolymath/panic-attack#32.
